# 바이브 코딩 준비 프로세스 (마스터)

> 바이브 코딩을 시작하기 전, 아이디어를 검증하고 서비스를 구체화하는 표준 프로세스  
> **제작자**와 **Claude**의 협업만으로 모든 단계를 완성한다

---

## 이 프로세스의 목적

아이디어를 바이브 코딩으로 바로 넘길 수 있는 상태로 만드는 것.  
구체적으로는 다음 3가지 문서를 완성하는 것이다.

| 문서 | 역할 | 주요 독자 |
|---|---|---|
| **서비스 기획서** | 왜 만드는가, 무엇을 만드는가 | 제작자 |
| **PRD** | 기능 명세, 화면 구성, 기술 스택 | Claude (AI) |
| **액션플랜** | Phase별 구현 순서와 체크리스트 | Claude (AI) |

---

## 프로세스 선택

아이디어의 구체화 수준과 목적에 따라 두 가지 프로세스 중 하나를 선택한다.

```
아이디어가 있다
        ↓
   어떤 상태인가?
        │
        ├─── 아이디어가 구체적이고              ──→  간편 프로세스
        │    검증이 필요 없는 경우                    (app-build-process-prd 참조)
        │    (학습용, 빠른 프로토타입 등)
        │
        └─── 실제 출시를 목표로 하거나          ──→  고급 프로세스
             아이디어/타겟/시장이                     (이 문서)
             아직 불명확한 경우
```

### 간편 프로세스
`app-build-process-prd.md` 참조  
Claude와의 기획 인터뷰 → PRD + 액션플랜 생성 → 바이브 코딩 시작  
예상 소요 시간: **1~2시간**

### 고급 프로세스 (이 문서)
4단계 검증 및 구체화 → 서비스 기획서 + PRD + 액션플랜 생성 → 바이브 코딩 시작  
예상 소요 시간: **4~8시간** (단계별 1~2시간)

---

## 고급 프로세스: Claude 협업 방식

이 프로세스의 모든 단계는 **제작자와 Claude의 대화**로 진행된다.  
실제 사용자 인터뷰나 외부 조사 없이, Claude가 시장 지식을 바탕으로  
가상의 결과를 생성하고 제작자가 방향을 결정하는 방식이다.

| 역할 | 책임 |
|---|---|
| **Claude** | 페르소나 생성, 가상 인터뷰 시뮬레이션, 경쟁사 분석, 문서 초안 작성 및 검토 |
| **제작자** | 방향 결정, 내용 수정 및 확정, 최종 판단 |

### 제작자의 선택을 돕는 원칙

제작자가 방향을 결정해야 하는 모든 시점에서 Claude는 다음 원칙을 따른다.

- **최소 3개의 안**을 구체적인 근거와 함께 제시한다
- **1개의 안을 추천**하고 추천 이유를 명시한다
- 제작자는 제시된 안 중 선택하거나, 수정을 요청하거나, 새로운 방향을 제안할 수 있다

이 원칙이 적용되는 주요 시점:

| 단계 | 선택 시점 |
|---|---|
| 1단계 | 타겟 페르소나 확정, 문제 정의 확정 |
| 2단계 | 포지셔닝 방향 확정 |
| 3단계 | 서비스 컨셉 확정, 핵심 기능 범위 확정 |
| 4단계 | 바이브 코딩 플랫폼 선택 |

---

## 전체 흐름

```
1단계                2단계               3단계               4단계
아이디어 검증   →   경쟁사 분석    →   서비스 기획    →   문서 완성
(약 1~2시간)        (약 1~2시간)        (약 1~2시간)        (약 1~2시간)
      ↓                  ↓                  ↓                  ↓
검증 결과 문서  →   경쟁사 분석표  →   서비스 기획서  →   PRD
(다음 단계 입력)    (다음 단계 입력)    (다음 단계 입력)    액션플랜
                                                            플랫폼 설정 파일
```

> 각 단계의 산출물은 다음 단계의 입력값이 된다.  
> 새 단계를 시작할 때 이전 단계 산출물을 Claude에게 함께 제공한다.

---

## 단계별 개요

### 1단계: 아이디어 검증
> "이 문제가 실제로 존재하는가? 사람들이 해결책을 원하는가?"

**예상 소요 시간:** 1~2시간

**Claude가 하는 일**
- Does-Solves-Who 프레임으로 잠재 타겟 그룹을 도출한다
- 타겟 그룹별 페르소나 3개를 생성하고 1개를 추천한다
- 확정된 페르소나를 대상으로 가상 인터뷰 결과를 시뮬레이션한다
- 문제의 실재 여부와 심각도를 평가한다

**제작자가 하는 일**
- 페르소나 3개 중 1개를 선택(또는 수정)하여 확정한다
- 가상 인터뷰 결과를 검토하고 Go / No-Go를 결정한다

**산출물:** 검증 결과 문서
- 확정된 페르소나
- 가상 인터뷰 요약
- 문제 정의 (한 문장)
- Go / No-Go 결정 및 근거

**No-Go 시 처리**
- 문제 정의가 잘못된 경우 → 문제를 재정의하고 1단계 재시작
- 타겟이 잘못된 경우 → 페르소나를 교체하고 1단계 재시작
- 아이디어 자체가 가망 없는 경우 → 새로운 아이디어로 처음부터 시작

---

### 2단계: 경쟁사 분석
> "이미 이걸 잘 하고 있는 곳이 있는가? 우리가 들어갈 자리가 있는가?"

**예상 소요 시간:** 1~2시간  
**입력값:** 1단계 검증 결과 문서

**Claude가 하는 일**
- 직접 경쟁사와 간접 경쟁사를 분류하여 조사한다
- 7가지 기준(성장성, 규모, 자금, 비용 지출, 고객 정서, 마케팅 전략, 전문성)으로 평가표를 작성한다
- 포지셔닝 방향 3가지를 제안하고 1가지를 추천한다

**제작자가 하는 일**
- 경쟁사 분석 결과를 검토한다
- 포지셔닝 방향 3가지 중 1가지를 선택(또는 수정)하여 확정한다
- Go / No-Go를 결정한다

**산출물:** 경쟁사 분석표 + 포지셔닝 메모
- 경쟁사 목록 및 평가표
- 확정된 포지셔닝 방향
- Go / No-Go 결정 및 근거

**No-Go 시 처리**
- 압도적인 경쟁자가 있지만 차별화 가능한 경우 → 포지셔닝을 바꾸고 2단계 재시작
- 차별화가 불가능한 경우 → 새로운 아이디어로 1단계부터 시작

---

### 3단계: 서비스 기획
> "검증된 문제를 어떤 서비스로 해결할 것인가?"

**예상 소요 시간:** 1~2시간  
**입력값:** 검증 결과 문서 + 경쟁사 분석표

**Claude가 하는 일**
- 1~2단계 결과를 종합하여 서비스 컨셉 3가지를 제안하고 1가지를 추천한다
- 확정된 컨셉을 바탕으로 서비스 기획서 초안을 작성한다
  - 핵심 가치 한 문장
  - 기능 범위 (Must Have / Nice to Have / Out of Scope)
  - 주요 화면 흐름
  - 성공 지표

**제작자가 하는 일**
- 서비스 컨셉 3가지 중 1가지를 선택(또는 수정)한다
- 서비스 기획서 초안을 검토하고 수정·확정한다

**산출물:** 서비스 기획서
- 서비스 개요 및 핵심 가치
- 타겟 사용자 (페르소나 기반)
- 확정된 기능 목록
- 주요 화면 흐름
- 성공 지표

---

### 4단계: 문서 완성 (빌드 준비)
> "Claude가 바로 구현을 시작할 수 있는 상태인가?"

**예상 소요 시간:** 1~2시간  
**입력값:** 서비스 기획서

**Claude가 하는 일**
- 서비스 기획서를 바탕으로 PRD를 작성한다
- PRD를 바탕으로 액션플랜을 작성한다
- 문서 전체를 검토하고 모순·누락을 확인한다
- 바이브 코딩 플랫폼 3가지를 비교하고 1가지를 추천한다
- 선택한 플랫폼에 맞는 기본 설정 파일을 생성한다

**제작자가 하는 일**
- PRD와 액션플랜을 최종 확인한다
- 바이브 코딩 플랫폼을 선택한다

**산출물:** PRD + 액션플랜 + 플랫폼 설정 파일

## 문서 체계

이 프로세스는 GitHub 저장소로 관리하며, Claude 프로젝트 지식에 연결하여 사용한다.

```
바이브 코딩 준비 프로세스 (마스터)        ← 이 문서
│
├── 단계별 가이드
│   ├── 1단계 가이드: 아이디어 검증
│   ├── 2단계 가이드: 경쟁사 분석
│   ├── 3단계 가이드: 서비스 기획
│   └── 4단계 가이드: 문서 완성
│
├── 단계별 프롬프트
│   ├── 1단계 프롬프트 모음
│   ├── 2단계 프롬프트 모음
│   ├── 3단계 프롬프트 모음
│   └── 4단계 프롬프트 모음
│
└── 보조 문서 (템플릿)
    ├── 검증 결과 문서 템플릿
    ├── 경쟁사 분석표 템플릿
    ├── 서비스 기획서 템플릿
    ├── 페르소나 템플릿
    └── 가상 인터뷰 템플릿
```
