# 1단계: 아이디어 검증 프롬프트 모음

> 이 파일은 아이디어 검증 단계에서 Claude에게 사용하는 프롬프트 모음이다.  
> 각 프롬프트는 순서대로 사용하도록 설계되었다.

---

## 프롬프트 목록

| 코드 | 프롬프트 이름 | 사용 시점 |
|---|---|---|
| P-01 | 문제 구체화 | 아이디어를 처음 꺼낼 때 |
| P-02 | 페르소나 생성 | 문제가 정의된 후 |
| P-03 | 가상 인터뷰 진행 | 페르소나 확정 후 |

| P-05 | No-Go 방향 재설정 | No-Go 판정 시 |

## [P-01] 문제 구체화

> **목적:** 막연한 아이디어를 "실제로 존재하는 문제"로 만든다.

```
나는 지금 아이디어를 검증하고 싶어.
Does-Solves-Who 프레임을 사용해서 내 아이디어를 분석해줘.

내 아이디어: [여기에 아이디어를 자유롭게 적어줘]

아래 순서로 진행해줘:

1. 내 아이디어에서 핵심 문제를 찾아줘
   - 문제를 구체적인 상황으로 재표현해줘 (누가, 언제, 어떤 상황에서 겪는지)
   - 이 문제의 심각도를 낮음/중간/높음으로 평가하고 이유를 설명해줘

2. 기존 해결 방법을 3가지 이상 나열해줘
   - 각 방법의 한계점도 함께 써줘

3. 내 아이디어가 기존 방법보다 나은 이유를 분석해줘
   - 차별점이 명확하면 계속 진행
   - 차별점이 불명확하면 솔직하게 말해줘

4. 잠재적 타겟 그룹을 2~3개 제안해줘
   - 이 문제를 가장 심각하게 겪는 사람들 중심으로
```

---

## [P-02] 페르소나 생성

> **목적:** 타겟을 구체적인 "사람"으로 만든다.

```
앞에서 정의한 문제와 타겟 그룹을 바탕으로
페르소나 3개를 만들어줘.

각 페르소나에는 다음 항목이 포함되어야 해:
- 이름, 나이, 직업
- 생활 패턴 (일정, 주요 활동, 기기 사용 습관)
- 이 서비스와 관련된 구체적인 상황
- 현재 해결 방법과 그 한계
- 핵심 불편함 (감정 언어로 표현)
- 이 서비스를 사용한다면 가장 기대하는 것

3개를 만든 후:
- 각 페르소나의 특징을 한 줄로 요약해줘
- 1단계 검증에 가장 적합한 페르소나 1개를 추천하고, 이유를 설명해줘
- 내가 확정할 때까지 다음 단계로 넘어가지 말아줘
```

---

## [P-03] 가상 인터뷰 진행

> **목적:** 페르소나가 이 서비스를 실제로 사용할지 시뮬레이션한다.

```
지금부터 확정된 페르소나 [페르소나 이름]으로 가상 인터뷰를 진행할게.

[옵션 A — Claude가 인터뷰어와 응답자를 모두 진행]
확정된 페르소나 기준으로 아래 5가지 질문에 대한 답변을 작성해줘.
답변은 페르소나의 말투와 심리를 반영해서 현실적으로 써줘.
답변이 끝나면 인터뷰 요약과 핵심 인사이트를 정리해줘.

[옵션 B — 내가 직접 인터뷰할 때]
지금부터 너는 [페르소나 이름]이야.
내가 질문하면 페르소나의 입장에서 대답해줘.
답변은 자연스럽고 현실적으로. 단, 인터뷰가 끝나면
가상 결과임을 명시하고 핵심 인사이트를 요약해줘.

인터뷰 질문:
1. 이런 상황(문제)을 실제로 겪어본 적 있나요?
2. 지금은 어떻게 해결하고 있나요?
3. 기존 방법에서 가장 불편한 점은 뭔가요?
4. 이런 서비스가 있다면 사용할 것 같나요? 솔직하게 말해줘요.
5. 사용하지 않는다면, 가장 큰 이유가 뭔가요?

⚠️ 이 인터뷰는 Claude가 생성한 가상 결과입니다.
실제 사용자 인터뷰를 대체할 수 없으며, 방향 판단의 참고 자료로 사용합니다.
```

## [P-04] Go / No-Go 판단

> **목적:** 지금까지의 검증 내용을 종합해서 진행 여부를 판단한다.

```
지금까지의 아이디어 검증 내용을 종합해서 Go/No-Go를 판단해줘.

판단 기준:
- 문제가 구체적인 상황으로 표현되었는가
- 타겟 페르소나가 이 문제를 실제로 겪는다고 판단되는가
- 기존 해결 방법 대비 명확한 차별점이 있는가
- 가상 인터뷰에서 페르소나의 반응이 긍정적이었는가
- 타겟 규모가 너무 좁지 않은가

판단 결과를 아래 형식으로 정리해줘:

## 검증 요약
- 문제 정의: (한 문장)
- 핵심 타겟: (한 줄)
- 차별점: (한 줄)
- 인터뷰 반응: 긍정 / 중립 / 부정

## 강점
(3가지 이상)

## 우려 사항
(솔직하게)

## 판정
**Go** 또는 **No-Go**
(판정 이유를 2~3문장으로)
```

---

## [P-05] No-Go 방향 재설정

> **목적:** No-Go 판정을 받은 후 다음 방향을 결정한다.

```
No-Go 판정이 나왔어.
원인을 분석하고, 3가지 재설정 방향을 제안해줘.

분석 항목:
- No-Go의 주요 원인 (문제 불명확 / 타겟 과소 / 기존 대안 충분 / 기타)
- 핵심 약점이 무엇인지 솔직하게

재설정 방향 3가지:
각 방향마다:
- 방향 이름
- 구체적인 변경 내용
- 이 방향의 장단점
- 예상 소요 시간

마지막에 1가지를 추천하고, 명확한 이유를 설명해줘.
내가 선택을 결정할 때까지 진행하지 말아줘.
```
