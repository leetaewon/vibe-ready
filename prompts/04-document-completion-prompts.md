## [P-01] PRD 생성

> **목적:** 서비스 기획서를 AI 코딩 도구가 읽을 수 있는 PRD로 변환한다.

```
3단계에서 완성한 서비스 기획서를 바탕으로 PRD를 작성해줘.

[서비스 기획서 내용]
(templates/03-service-plan.md 전체 내용을 여기에 붙여넣기)

PRD 형식:

# PRD: [서비스 이름]

## 1. 앱 개요
(서비스 목적을 2~3문장으로. 처음 보는 사람도 바로 이해할 수 있게)

## 2. 타겟 사용자
(페르소나 기반으로 구체적으로. 사용자 유형이 여럿이면 각각 서술)

## 3. 핵심 기능
(Must Have 기능 목록. 번호 목록, 기능당 1~2줄 설명)

## 4. 제외 기능 (Out of Scope)
(Won't Have 목록. AI가 임의로 추가하지 않도록 이유 포함)

## 5. 화면 구성
(전체 화면 목록. 화면명 + 역할 + 주요 요소)

## 6. 데이터 구조
(저장해야 할 정보. 테이블명과 주요 필드 수준으로)

## 7. 기술 스택
(확정된 기술과 외부 서비스. 불확실하면 비워두거나 후보를 적어줘)

## 8. 디자인 가이드
(색상, 폰트, 분위기, 톤 앤 매너. 참고 레퍼런스가 있으면 포함)

PRD를 먼저 보여줘. 내가 확인하고 컨펌하면 액션플랜을 만들어줘.
```

## [P-02] 액션플랜 생성

> **목적:** PRD를 기반으로 Phase별 구현 순서를 체크리스트로 만든다.

```
확인했어. 이제 액션플랜을 만들어줘.

위 PRD를 기반으로 아래 형식에 맞게 액션플랜을 작성해줘.

# ActionPlan: [서비스 이름]

## Phase 1. 프로젝트 초기 설정
- [ ] (폴더 구조 생성)
- [ ] (패키지 / 의존성 설치)
- [ ] (환경 변수 설정)
- [ ] (기본 라우팅 구성)

## Phase 2. 데이터 구조 설계
- [ ] (DB 스키마 또는 데이터 모델 정의)
- [ ] (주요 타입/인터페이스 정의)

## Phase 3. UI 컴포넌트 구현
- [ ] (공통 레이아웃)
- [ ] (공통 컴포넌트)
- [ ] (주요 화면 기본 구조)

## Phase 4. 핵심 기능 구현
(Must Have 기능을 의존성 순서에 맞게 나열)
- [ ] (기능 1)
- [ ] (기능 2)
...

## Phase 5. 테스트 및 배포
- [ ] (기본 기능 테스트)
- [ ] (배포 설정)
- [ ] (최종 확인)

작성 원칙:
- 각 Phase는 독립적으로 완료 가능해야 한다
- 체크리스트 항목은 하나의 완결된 작업 단위로
- Phase 4의 기능 순서는 의존성 높은 것 → 낮은 것 순서로
```

## [P-03] QA 검토

> **목적:** 완성된 PRD와 액션플랜을 빌드 전에 점검한다.

```
방금 만든 PRD와 액션플랜을 바이브 코딩을 시작하기 전에 검토해줘.

확인 항목:
1. MVP 범위가 적절한가? (너무 많거나 적지 않은가)
2. 선택 예정인 플랫폼으로 구현 가능한 수준인가?
3. 기술 스택이 기능 요구사항에 맞는가?
4. 빠진 항목이나 서로 모순된 내용이 있는가?
5. 액션플랜의 Phase 순서가 논리적인가?
6. 제외 기능이 명확하게 명시되어 있는가?

문제가 있으면 수정 제안을 구체적으로 해줘.
없으면 "QA 통과"라고 알려줘.
```

## [P-04] 플랫폼 선택

> **목적:** 서비스 특성과 제작자 상황에 맞는 바이브 코딩 플랫폼을 고른다.

```
우리 서비스에 맞는 바이브 코딩 플랫폼 3가지를 추천해줘.

서비스 정보:
- 서비스 유형: [웹앱 / 모바일앱 / 대시보드 등]
- 주요 기능: [PRD의 핵심 기능 요약]
- 기술 스택: [PRD의 기술 스택]
- 배포 목표: [프로토타입 공유 / 실제 출시 / 내부 사용]
- 제작자 상황: [코딩 경험 없음 / 약간 있음 / 개발자]

각 플랫폼마다:
- 플랫폼 이름
- 이 서비스에 적합한 이유
- 주의할 점이나 한계
- 시작 방법 한 줄 요약

3가지 제안 후 1가지를 추천하고, 명확한 이유를 설명해줘.
내가 선택을 결정할 때까지 다음 단계로 넘어가지 말아줘.
```

## [P-05] 시작 프롬프트 생성

> **목적:** 선택한 플랫폼에서 바로 사용할 수 있는 첫 번째 프롬프트를 만든다.

```
플랫폼 [선택한 플랫폼 이름]으로 확정할게.

이 플랫폼에서 바이브 코딩을 시작할 때 사용할
첫 번째 프롬프트를 작성해줘.

포함할 내용:
- PRD와 액션플랜 파일을 첨부한다는 안내
- Phase 1부터 순서대로 진행해달라는 지시
- 각 Phase가 끝나면 알려달라는 지시
- 이 서비스에 특별히 주의해야 할 사항

그리고 Phase 전환 시 사용할 프롬프트 템플릿도 만들어줘:
"Phase [N] 확인했어. [확인 사항]. Phase [N+1]로 넘어가줘."
```

## [P-06] CLAUDE.md 생성 *(Cursor 등 사용 시)*

> **목적:** Cursor 등 Claude 기반 도구에 프로젝트 컨텍스트를 제공하는 파일을 만든다.

```
이 프로젝트의 CLAUDE.md 파일을 만들어줘.

CLAUDE.md는 Claude Code 또는 Cursor가 이 저장소를 처음 열었을 때
프로젝트를 올바르게 이해하기 위한 안내 문서야.

포함할 내용:
1. 프로젝트 개요 (서비스 한 줄 정의)
2. 기술 스택 및 주요 의존성
3. 폴더 구조 설명
4. 개발 시 지켜야 할 규칙
   - 코드 스타일
   - 컴포넌트 작성 방식
   - 커밋 메시지 형식 등
5. 자주 사용하는 명령어
6. 주의사항 (절대 하면 안 되는 것)

PRD와 액션플랜 내용을 기반으로 이 프로젝트에 맞게 작성해줘.
```
