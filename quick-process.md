# 간편 프로세스: 빠르게 시작하는 바이브 코딩

> Claude / Gemini → PRD + ActionPlan → AI Studio Antigravity  
> 막연한 아이디어를 체계적인 문서로 만들고, 그 문서로 앱을 빌드하는 4단계 프로세스

---

## 전체 흐름

```
STEP 1              STEP 2                  STEP 3              STEP 4
기획 인터뷰    →    PRD + ActionPlan    →    AI 자동 검토    →    AI Studio 빌드
(Claude가 질문)     생성 (Claude)            (Claude)             (Antigravity)
```

---

## STEP 1. 기획 인터뷰 — Claude/Gemini에서

> 혼자 막연하게 기획하는 대신, Claude가 올바른 질문을 던져서 기획을 완성시킨다.

### 📋 프롬프트 (복붙용)

```
나는 앱을 기획 중이야.
아래 항목들을 순서대로 하나씩 질문해줘.
내 답변을 받은 후 다음 질문으로 넘어가.

1. 앱의 핵심 목적 (한 문장으로)
2. 주요 사용자가 누구인지
3. 반드시 있어야 할 기능 (Must Have)
4. 있으면 좋지만 없어도 되는 기능 (Nice to Have)
5. 데이터 저장이 필요한지, 필요하다면 어떤 데이터인지
6. 로그인/회원가입이 필요한지
7. 관리자 기능이 필요한지
8. 디자인 분위기나 참고할 레퍼런스
9. 공유 링크로 배포할 건지, 로컬에서만 쓸 건지

모든 질문이 끝나면 답변을 정리해서 보여줘.
```

### ✅ 이 단계의 결과물
- 9개 항목이 채워진 **기획 요약본**

---

## STEP 2. PRD + ActionPlan 생성 — Claude/Gemini에서

> 기획 요약본을 바탕으로 AI Studio 빌드에 필요한 두 가지 문서를 만든다.  
> PRD를 먼저 확인하고 컨펌한 후 ActionPlan을 생성한다.

### 📋 프롬프트 (복붙용)

```
위 내용을 바탕으로 아래 형식에 맞게
PRD와 ActionPlan을 순서대로 만들어줘.
PRD를 먼저 보여주고, 내가 컨펌하면 ActionPlan을 만들어줘.

[PRD 형식]
# PRD: 앱 이름
## 1. 앱 개요 (2~3문장)
## 2. 타깃 사용자
## 3. 핵심 기능 (번호 목록)
## 4. 제외 기능 (Out of Scope)
## 5. 화면 구성 (페이지별)
## 6. 데이터 구조 (저장할 정보)
## 7. 기술 스택
## 8. 디자인 가이드

[ActionPlan 형식]
# ActionPlan: 앱 이름
## Phase 1. 프로젝트 초기 설정
## Phase 2. 데이터 구조 설계
## Phase 3. UI 컴포넌트 구현
## Phase 4. 핵심 기능 구현
## Phase 5. 테스트 및 배포
(각 Phase는 체크리스트 형식으로)
```

### ✅ 이 단계의 결과물
- `PRD.md` — 앱 명세서
- `ActionPlan.md` — Phase별 체크리스트

---

## STEP 3. AI 자동 검토 — Claude/Gemini에서

> 잘 모르는 사람도 AI가 대신 QA를 해준다.  
> 문제가 있으면 수정 제안을 받고, 괜찮으면 AI Studio용 첫 프롬프트를 받는다.

### 📋 프롬프트 (복붙용)

```
방금 만든 PRD와 ActionPlan을
Google AI Studio에서 빌드하기 전에 검토해줘.

확인 항목:
1. MVP 범위가 적절한가? (너무 많거나 적지 않은가)
2. AI Studio + Antigravity로 구현 가능한 수준인가?
3. 기술 스택이 앱 요구사항에 맞는가?
4. 빠진 항목이나 서로 모순된 내용이 있는가?
5. ActionPlan의 Phase 순서가 논리적인가?

문제가 있으면 수정 제안을 해주고,
괜찮으면 AI Studio에 넣을
첫 번째 프롬프트를 작성해줘.
```

### ✅ 이 단계의 결과물
- 검토 리포트 + 수정사항 (있을 경우)
- **AI Studio용 첫 번째 프롬프트** (바로 복붙 가능)

---

## STEP 4. AI Studio 빌드 — Google AI Studio (Antigravity)

> PRD.md + ActionPlan.md 파일을 첨부하고 빌드를 시작한다.  
> Phase 단위로 진행하며 각 단계를 확인하면서 넘어간다.

### 📋 시작 프롬프트 (복붙용)

```
첨부한 PRD와 ActionPlan을 기반으로 앱을 만들어줘.
ActionPlan의 Phase 1부터 순서대로 진행해줘.
각 Phase가 끝나면 나에게 알려줘.
```

### Phase 전환 프롬프트

```
Phase [N] 확인했어. Phase [N+1]로 넘어가줘.
```

### ✅ 이 단계의 결과물
- 완성된 앱
- 공유 가능한 링크 (Share 버튼)

---

## 왜 이 방식인가?

| 일반 방식 | PRD 기반 방식 |
|---|---|
| "습관 관리 앱 만들어줘" | 기획 → PRD → ActionPlan → 빌드 |
| AI가 마음대로 해석 | 문서 기반으로 정확하게 구현 |
| 수정이 많고 방향이 흔들림 | Phase별 컨트롤 가능 |
| 남의 프롬프트 복붙 | 내 앱에 맞는 문서 직접 생성 |
| 설계와 빌드가 뒤섞임 | 설계 완료 후 빌드에만 집중 |
